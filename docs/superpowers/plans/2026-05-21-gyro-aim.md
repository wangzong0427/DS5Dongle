# Gyro Aim Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add optional L2-gated gyro aiming by mapping DualSense gyro motion onto the right stick.

**Architecture:** Add a focused `gyro_aim` module that processes the 63-byte input report before it is sent to the host. Extend firmware config with safe defaults and validation. Keep USB descriptors and Bluetooth connection flow unchanged.

**Tech Stack:** C++20 firmware, Raspberry Pi Pico SDK, TinyUSB, BTstack, host-side C++ tests compiled with the local compiler.

---

## File Structure

- Create `src/gyro_aim.h`: public processor API for report mutation.
- Create `src/gyro_aim.cpp`: pure gyro mapping implementation and smoothing state.
- Create `tests/gyro_aim_test.cpp`: host-buildable assertions for the processor.
- Modify `src/config.h`: add gyro config fields.
- Modify `src/config.cpp`: validate new fields and default them safely.
- Modify `src/main.cpp`: call the processor before copying BT input reports to USB report storage.
- Modify `CMakeLists.txt`: include `src/gyro_aim.cpp` in firmware target.
